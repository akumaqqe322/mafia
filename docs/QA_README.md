# QA README — Telegram Mafia Bot

**Для инструкций по установке и техническому запуску проекта см. основной [README.md](../README.md).**

## 1. Назначение документа
Этот файл предназначен для ручного тестирования Telegram-бота Mafia. Он содержит практические инструкции и чеклисты для проверки основных игровых механик, сценариев взаимодействия и работы панели администратора.

Документ не является пользовательской инструкцией для финальных игроков, а служит инструментом для обеспечения качества (QA) текущей сборки.

---

## 2. Что нужно перед тестом (Prerequisites)

- **Telegram-группа или супергруппа:** Бот должен быть добавлен в группу.
- **Права бота:** Бот должен иметь разрешение на отправку и редактирование своих сообщений.
- **Личные сообщения (ЛС):** Игроки **должны сначала открыть чат с ботом** и нажать кнопку "Start" в ЛС, иначе бот не сможет отправить им роль или меню действий.
- **Участники:** Для режима `classic_5_6` требуется минимум 5 живых участников.
- **Роли тестировщиков:**
    - Минимум один аккаунт должен быть **Admin/Owner** группы в Telegram (для тестов панели управления).
    - Минимум один аккаунт должен быть обычным участником группы.

## 3. Быстрый smoke test

1. **Запуск лобби:** В группе отправить команду `/game`.
   - **Ожидаемо:** Появляется карточка лобби. Есть кнопки «Вступить» (Join) и «Отмена» (Cancel).

2. **Вступление:** Игроки нажимают кнопку «Вступить».
   - **Ожидаемо:** Игрока перекидывает в ЛС с ботом. Бот подтверждает вступление. Карточка в группе обновляется (список игроков растет).

3. **Старт:** Создатель игры или Админ нажимает «Старт».
   - **Ожидаемо:** Игра начинается. В ЛС игрокам приходят их роли. В группе появляется сообщение о начале Первой Ночи.

4. **Дневная фаза:** Подождать или форсировать переход в день (см. раздел 5).
   - **Ожидаемо:** Бот сообщает о наступлении дня.

5. **Голосование:** Дойти до фазы голосования.
   - **Ожидаемо:** Появляется панель голосования. Живые игроки могут нажать кнопку за кандидата.

6. **Финал:** Завершить игру (победа одной из сторон или Admin Force Finish).
   - **Ожидаемо:** Бот отправляет итоговый отчет. Роли всех игроков раскрываются.

## 4. Проверка обычного игрового цикла

### 4.1 Создание лобби
- Проверить: Команда `/game` создает только одно лобби в чате. Повторный вызов не должен плодить дубликаты (старое должно отменяться или игнорироваться).

### 4.2 Вступление игроков
- Проверить: Игрок может вступить только один раз.
- Проверить: Вступление после начала игры (через старую ссылку) должно быть отклонено сообщением "Игра уже началась".

### 4.3 Старт игры
- Проверить: Обычный игрок (не создатель и не админ) **не может** запустить игру.
- Проверить: Создатель может запустить игру.
- Проверить: Админ группы может запустить игру, даже если он не создатель.
- Проверить: Игроки получили роли в ЛС. Роли соответствуют выбранному пресету.

### 4.4 Ночные действия
- Проверить: Мафия получает меню выбора жертвы.
- Проверить: Шериф получает меню проверки.
- Проверить: Доктор получает меню лечения.
- Проверить: Старые кнопки (из прошлой ночи) не работают.

### 4.5 День
- Проверить: Сообщение о смерти корректно указывает игрока.
- Проверить: Если ночью никто не погиб, бот сообщает, что ночь прошла спокойно / все жители проснулись.
- Проверить: Роли убитых/проверенных **не раскрываются** до конца игры (если это не предусмотрено настройками).

### 4.6 Голосование
- Проверить: Голосовать могут только живые игроки.
- Проверить: Нельзя голосовать за самого себя.
- Проверить: После окончания таймера кнопки удаляются/отключаются.

### 4.7 Финал
- Проверить: Роли раскрыты у всех (и живых, и мертвых).
- Проверить: Кнопки старой игры полностью недействительны.
- Проверить: Можно сразу создать новое лобби через `/game`.

## 5. Проверка Панели Администратора (Admin Panel)

Панель вызывается командой `/admin_game` в чате группы.

### 5.1 Доступ
- **Сценарий A:** Обычный пользователь пишет `/admin_game` -> Доступ запрещен.
- **Сценарий B:** Telegram Admin/Owner пишет `/admin_game` -> Панель открывается.
- **Сценарий C:** Создатель игры (не админ группы) пишет `/admin_game` -> Доступ запрещен (панель только для админов чата).

### 5.2 Обновление (Refresh)
- Шаги: Изменить что-то в лобби (добавить игрока), нажать Refresh в панели.
- **Ожидаемо:** Счётчик игроков и версия (v) в панели обновились.

### 5.3 Исключение из лобби (Kick)
- Шаги: Лобби активно. Открыть `/admin_game` -> кнопка списка игроков / кика -> Выбрать игрока.
- **Ожидаемо:** Игрок удален из лобби. Карточка лобби в группе обновилась. Игрок может вступить заново.

### 5.4 Принудительный шаг (Force Tick)
- Шаги: Идет ночь. Админ нажимает "Завершить фазу".
- **Ожидаемо:** Игра немедленно переходит в день (или следующую фазу). Старые таймеры PhaseWorker не должны конфликтовать. В LOBBY и FINISHED действие должно быть отклонено.

### 5.5 Принудительное завершение (Force Finish)
- Шаги: Идет активная фаза. Админ нажимает "Завершить игру".
- **Ожидаемо:** Появляется экран подтверждения (**Confirmation screen**). Игра еще НЕ завершена.
- Нажать "Назад" -> Возврат в админку, игра продолжается.
- Нажать "Да, остановить" -> Игра завершается. В чат летит финальный отчёт. После завершения можно создать новое лобби через /game.

## 6. Проверка "протухших" кнопок (Stale Callbacks)

1. Открыть панель администратора.
2. Сделать действие в другом месте (например, начать игру или подождать тика).
3. Нажать кнопку в уже неактуальной панели.
- **Ожидаемо:** Бот пишет "Панель устарела" или "Игра уже завершена". Действие не срабатывает. Это критично для предотвращения Race Conditions.

---

## 7. Проверка граничных случаев Telegram

- **Бот заблокирован:** Один из игроков заблокировал бота перед выдачей ролей. Бот должен обработать ошибку (логировать) и, по возможности, уведомить чат, что игроку нельзя отправить роль.
- **Удаление сообщений:** Удаление сообщения с лобби или голосованием не должно вешать движок игры.
- **Double Click:** Быстрое нажатие на одну кнопку двумя админами. Один должен получить успех, второй — уведомление об устаревшем состоянии.

## 8. Чеклист перед релизом

- [ ] Бот в группе и имеет права на сообщения.
- [ ] Тестировщик-админ и тестировщик-игрок готовы.
- [ ] `/admin_game` не пускает простых смертных.
- [ ] Kick из лобби работает и чистит все привязки.
- [ ] Night actions работают (приходят в ЛС, цели выбираются).
- [ ] Результаты ночи логичны.
- [ ] Голосование отрабатывает корректно.
- [ ] Force Finish требует подтверждения.
- [ ] Финальный отчет показывает роли на русском.
- [ ] После финала можно начать новую игру без ошибок.

## 9. Известные ограничения (Known Limitations)

- **Web-Admin:** Внешняя веб-панель находится в стадии MVP и не является основным инструментом тестируемого функционала.
- **Audit Log:** Отсутствует аудит действий — нельзя точно узнать из логов бота (в Telegram), какой именно администратор нажал кнопку.
- **Force Actions:** Функции Force Tick и Force Finish проверяются вручную, автоматические интеграционные тесты для них ограничены из-за сложности моков.
- **Роли в админке:** Текстовая админка в Telegram не раскрывает роли живых игроков до финала игры (защита от «подглядывания»).
- **Кик из игры:** Реализован только кик из Лобби. Исключение игроков во время активной игры не поддерживается.
